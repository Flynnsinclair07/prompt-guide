# 7-day progress checklist

A working tour of the bundle, paced for someone actively job-hunting. One day = one stage of the search. Each day ties two or three prompts/templates to a concrete output you'll have at the end.

This isn't a strict prescription — if you have an offer in hand right now, jump to Day 7. If you're 3 weeks out and just starting to update materials, do Day 1–2 only and re-run the rest closer to interviews. The order is the order most searches actually go through; the days are markers, not deadlines.

## Day 1 — Resume

**Output**: a tailored, ATS-readable resume for one specific role.

- [ ] Pick the **one role** you'd be most excited about right now. Don't try to write a "general" resume — they always lose. Add the role to the **Job tracker** (Stage = Researching).
- [ ] Run **Module 1, Prompt 1 (Resume from scratch)** if you don't have a current resume, or **Prompt 2 (Resume rewrite)** if you do. Paste the actual JD and your raw experience.
- [ ] Run the output through **ATS optimization workflow** (Templates page) — 5-step audit. Fix anything that fails.
- [ ] Run **Prompt 3 (Bullet rewrite)** on your two weakest bullets, individually.
- [ ] Save the result with a versioned filename in the **Job tracker** Resume version column.

## Day 2 — Cover letter (if required)

**Output**: a cover letter you'd send for one specific role, or a decision that you don't need one.

- [ ] Check whether the role actually requires a cover letter. Most software / data / product roles don't. If not, skip to Day 3.
- [ ] If yes, run **Module 2, Prompt 1 (Hook-first opener)** to draft the first paragraph. The opener does 70% of the work.
- [ ] Run **Module 2, Prompt 2 (Mirror-the-JD)** to draft the body.
- [ ] Use the **"What hiring managers actually skim for" cheat sheet** (Templates page) as a 13-point pre-send check. If your draft fails 3+ signals, rewrite.
- [ ] Save and log in the **Job tracker** Cover letter? column.

## Day 3 — LinkedIn

**Output**: a LinkedIn profile that recruiters can find and want to message.

- [ ] Run **Module 4, Prompt 1 (Headline)** — under 220 characters, keyword-dense.
- [ ] Run **Module 4, Prompt 2 (About section)** — the 2,600 chars recruiters read after the headline.
- [ ] Run **Module 4, Prompt 4 (Skills section keywording)** — pick the 50 skills that matter for Boolean search.
- [ ] Walk the **14-item LinkedIn optimization checklist** (Templates page). Most of these are 30-second fixes.
- [ ] If you're open to recruiter outreach, toggle "Open to Work" (only-recruiters mode).

## Day 4 — Target list + outreach

**Output**: 5–10 specific roles in your tracker, plus 2–3 outbound DMs.

- [ ] Add 5–10 roles you're considering to the **Job tracker** (Stage = Bookmarked or Researching).
- [ ] Pick 2–3 where a warm or cold reach-out makes sense.
- [ ] For warm reach-outs: run **Module 4, Prompt 8 (Post-connection follow-up)** if you've already connected, or use the cold-DM prompt if you need to ask for a connection first.
- [ ] For cold-DMs: run **Module 4, Prompt 6 (Recruiter cold-DM)** — under 300 chars, specific, not begging.
- [ ] Send. Log the date in **Last contact** for each tracker row.

## Day 5 — Application + screen prep

**Output**: applications submitted for top 3 roles, plus prep for any phone screen you have scheduled.

- [ ] Submit applications for your top 3 roles. Update **Stage = Applied** in the tracker.
- [ ] If a screen is scheduled, run **Module 3, Prompt 1 (Behavioral questions by role)** to predict what they'll actually ask.
- [ ] Run **Module 3, Prompt 6 ("Why this company?" research)** — most-skipped question, easy edge to take.
- [ ] If salary will come up on the screen, run **Module 3, Prompt 8 (Salary screen)** so you have a calibrated answer ready (don't go in cold on this one).

## Day 6 — Interview rehearsal

**Output**: 3 strong stories rehearsed in STAR shape, scored, rewritten.

- [ ] Pick the 3 stories you'd want to land in any interview (the ones that demonstrate the highest-leverage parts of your background).
- [ ] Run **Module 3, Prompt 4 (STAR answer structuring)** on each.
- [ ] Run the **Answer-scoring prompt** (last item in Module 3) on your written answers. Have it grade them and rewrite the weak ones.
- [ ] Run the **AI-grilling rehearsal workflow** (Module 3) — 3 rounds: warm-up, pressure, curveballs. The pressure round is where you'll hear yourself fold.
- [ ] If you're doing a panel or onsite, run **Prompt 9 (Panel / multi-interviewer prep)** to map who-asks-what.
- [ ] Before any interview, run **Prompt 11 (Reverse-interview)** — questions that make you look sharp AND surface red flags.

## Day 7 — Negotiation prep

**Output**: a defensible counter-offer ready to send when the offer arrives. (If you don't have an offer yet, this is dry-run prep — still worth it.)

- [ ] Fill out **Worksheet 1 (BATNA)** in the Templates page. Honest answers, not aspirational. This is the load-bearing input for everything else.
- [ ] Fill out **Worksheet 2 (Comp research)** — pull Levels.fyi, Glassdoor, Blind, plus 1–2 reference calls if you can swing them.
- [ ] Run **Module 5, Prompt 1 (Market-rate research)** with your role + level + city as inputs.
- [ ] When the offer arrives, run **Prompt 2 (Offer evaluation)** — decode the components before responding.
- [ ] Run **Prompt 3 (Counter-offer scripting)** with your BATNA and market data in hand.
- [ ] Use **Script 1 (Email: decline first offer politely and counter)** as the template for the actual email.
- [ ] Keep **Script 3 ("We don't have budget" rebuttal)** open in another tab — you'll need it.
- [ ] Update tracker: **Stage = Offer received**, log the comp range.

## After Day 7

- If you got an offer and signed: add it to the tracker as **Stage = Accepted** and update the changelog with what worked + what didn't (helpful for next time).
- If you're still in process on multiple roles: cycle through Days 5–7 per role until decisions land. Days 1–4 are setup work; you don't redo them per role.
- If everything stalled: that's information. Look at the tracker — where do most roles drop off? Stage = Applied with no response = resume / fit problem; Stage = Recruiter screen with no progress = positioning / pricing problem; Stage = Final round with no offers = closing problem. Each has a different fix in the bundle.
