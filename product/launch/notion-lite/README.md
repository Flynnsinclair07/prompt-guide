## Build steps

### Step 1 — Create the workspace
1. Log into Notion
2. Click "+ New page" in your sidebar
3. Title: `Job Hunter's Bundle — Companion Workspace`
4. Pick an icon (suggest: 💼 or 📚)
5. Pick a cover image (optional — can use the PDF cover screenshot)

### Step 2 — Add the parent pages
For each of the 4 parent pages above (Negotiation Scripts, Worksheets, LinkedIn Tools, Interview Tools):
1. Inside the workspace, type `/page` and select "Page"
2. Title it (e.g., "💰 Negotiation Scripts")
3. Hit Enter — opens the new page
4. Go back to the workspace homepage
5. Repeat for the other 3

### Step 3 — Paste each script/worksheet/tool as a sub-page
1. Open the parent page (e.g., "💰 Negotiation Scripts")
2. Type `/page` → "Page"
3. Title it (e.g., "Script 1 — Email: decline & counter")
4. Open the new page
5. Open the corresponding .md file in this directory (e.g., `script-01-email-decline-counter.md`)
6. Copy ALL the content (Cmd+A, Cmd+C in your editor)
7. Paste into the Notion page (Cmd+V)
8. Notion auto-converts markdown — code blocks, headers, lists all format correctly

Repeat for all 15 sub-pages.

### Step 4 — Convert worksheets to Notion databases
The 3 worksheet files have specific Notion database setup instructions in their headers. These are the highest-value Notion deliverables — buyers will use them on every offer evaluation.

**Worksheet 2 (Comp research)** — convert to a database with table view (see file for column setup)

**Worksheet 3 (Walkaway math)** — convert to a database with formula fields that auto-calculate (see file for formula setup — this is the marquee Notion feature)

### Step 5 — Set sharing
1. Top-right of the workspace page: click "Share"
2. Toggle on: "Share to web"
3. Set permission to "Can view" (read-only — buyers shouldn't be able to edit your master)
4. Copy the share URL
5. Paste this URL into the Gumroad listing's "Additional content" field (per `gumroad-listing.md`)

### Step 6 — Verify
- Open the share URL in an incognito browser (no Notion login)
- Confirm: all pages visible, all content readable, all databases interactive (the formulas should still work even for read-only viewers)
